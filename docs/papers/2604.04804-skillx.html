Abstract

现有 LLM agent 的"自我进化"范式存在三个结构性缺陷：孤立学习（agent 独立反复发现相似经验）、经验泛化弱（挖掘出的经验难以迁移）、模型能力瓶颈（提取质量受限于 agent 自身）。核心问题由此浮现：什么形式的经验可以跨越不同能力的 agent、跨越不同环境被广泛复用？

受 Anthropic Claude Skills 启发，SkillX 认为 skill 是更合适的抽象，但此前 skill 设计依赖长上下文渐进披露，对推理和沙箱要求过高。SkillX 通过 三层次技能设计 + 迭代精炼 + 探索式扩展，从 agent 轨迹中全自动构建可插拔技能知识库（SkillKB）：强模型（GLM-4.6）提炼的技能库可直接插拔到弱模型（Qwen3-32B），在 BFCL-v3 / AppWorld / τ²-Bench 上带来约 +10 分 性能提升。

Key Insight

经验 表示形式 的选择比经验 提取模型 的强弱更关键。层次化结构化经验（Planning → Functional → Atomic）支持跨模型迁移。

Framework Overview

SkillX 的自动化迭代 pipeline 集成 Skills Extraction（提取） / Skills Expansion（扩展） / Skills Refinement（精炼）三个阶段；技能库 $\mathcal{D} = S_{\text{plan}} \oplus S_{\text{func}} \oplus S_{\text{atomic}}$ 组织为 Planning / Functional / Atomic 三层。

Figure 2. SkillX 的自动化迭代 pipeline 与三层技能组织方式。

Planning Skills

任务级粒度。从成功轨迹压缩出有序高层步骤集合，过滤掉探索、回溯、试错等非必要转换。指导 functional skills 的组合方式。

Functional Skills

子任务级。可复用的、基于工具的子程序，对应子任务完成步骤。字段：name / document / content。对整体性能提升贡献最大。

Atomic Skills

单工具级。单个工具的增强规格说明：调用模式、参数配置、常见约束与失败模式。缺失时对关键 API 任务影响显著。

Rollout & Iterative Refinement

每轮 rollout 后，批次聚合通过精炼算子 $\phi$ 更新库：

$$\mathcal{D}^{(k+1)} = \mathcal{D}^{(k)} \cup \phi\!\left(\bigcup_{i=1}^{N_k} \mathcal{S}_i^{(k)}\right)$$

$\phi$ 包含：Skills Merge（余弦相似度聚类语义相近技能、过于复杂的合并结果分解为子技能）与 Skills Filter（General Filter 过滤外部依赖 / 过度封装；Tool-specific Filter 按环境 schema 验证）。最多迭代 3 轮。

Exploratory Expansion：种子数据稀缺时（如 AppWorld 数百 API），统计各工具使用情况，优先探索 under-explored 或 failure-prone 工具，从探索轨迹合成新任务后再次进入 pipeline——优于随机探索。

Analysis

哪种技能最有效？

Planning skills 对弱模型（Qwen3-32B、K2）一致减少执行步骤；直接加 Functional + Atomic 反而可能让弱模型过度模仿、损害性能。 Functional skills 对整体性能提升贡献最大。 Atomic skills 对关键 API 提供重要澄清，缺失时性能大幅下降。 最优组合因模型而异：GLM-4.6 = 全部；K2 = Functional + Atomic；Qwen3-32B = 仅 Planning。

迭代精炼 & 经验引导探索

多轮精炼在训练集和测试集均进一步提升性能，但数据有限时纯文本优化可能过拟合，需选合适轮数。 经验引导探索发现比随机探索更多样化的技能；在复杂环境、稀缺数据场景下，技能扩展是经验学习的关键组成。 执行效率上 SkillX 不追求最少步骤/最少 token，而是在效率与性能之间取得最佳平衡。

Usage 时的检索策略

先检索语义相似的 planning skills；LLM 结合结果改写任务特定 伪计划 $\tilde{p}(q)$，伪计划仅作检索 query 不注入 prompt（防止幻觉）；再以伪计划各步骤检索 functional & atomic skills，去重 + LLM 过滤后注入 system prompt。

跨环境泛化

SkillX 在 AppWorld（数百 API）、BFCL-v3（多轮函数调用）、τ²-Bench（对话式交互）三类风格迥异的 benchmark 均有效；进一步在 DeepSeek-V3.2、GPT-4.1 等强模型上同样带来稳定提升（Table 2）。

Takeaways

核心结论

  1. 层次化结构化经验（planning → functional → atomic）比单一形式更有利于跨模型迁移。
  2. 强模型提炼的 SkillKB 可直接插拔到弱模型，带来约 10% 的性能提升，同时提升执行效率。
  3. 经验 表示形式 比经验 提取模型 的强弱更关键。
  4. 探索式扩展 + 迭代精炼在数据稀缺的复杂环境中尤为重要。

局限性

  • 技能与具体工具 schema 绑定，跨域（不同工具生态）直接复用困难。
  • 当前主要针对工具调用型 agent，纯对话场景（无函数调用）覆盖不足。

对后续研究的启发

  1. 技能粒度分层是 agent 记忆系统设计的有效范式。
  2. 强模型蒸馏 → 弱模型受益，提供了低成本增强弱 agent 的实用路径。
  3. 经验引导探索显著优于随机探索。
  4. 伪计划改写作为检索中间桥梁，是缓解"任务 — 历史经验"语义不匹配的实用技巧。
