Abstract

Agent Skill 的生成长期依赖人工编写：领域专家手动撰写 Skill bundle，成本高昂且质量不稳定，难以规模化。更深层的问题是人机认知错位——人类按直觉编写的工作流与 LLM Agent 的推理执行逻辑存在结构性鸿沟，导致部分领域（如 Natural Science）中人工 Skill 反而使 Agent 性能下降。

EvoSkills 将 Skill Generator 与 Surrogate Verifier 置于协同演化循环中：Generator 迭代生成多文件 Skill bundle（SKILL.md + 可执行脚本 + 领域参考资料），Verifier 在信息隔离条件下独立合成测试断言并提供结构化诊断反馈，两者交替优化。核心创新在于无需 Ground-Truth 内容即可提供密集可操作的演化信号。在 SkillsBench（87 任务 / 11 领域）上，EvoSkills 以 71.1% pass rate 超越无 Skill 基线 +40.5 pp、超越人工精心编写的 Skill +17.6 pp，且演化 Skill 跨 6 个模型家族均迁移有效（+36～+44 pp）。

Key Insight

① 多文件 Skill bundle = SKILL.md + 可执行脚本 + 领域参考资料——比单一函数/API 结构复杂，一次生成不可靠，必须有闭环验证。
② Co-evolutionary Verification——Generator 与 Surrogate Verifier 协同演化，Verifier 自主合成测试断言作为代理奖励信号，彻底摆脱对 Ground-Truth 反馈内容的依赖；演化效率高（5 轮内收敛，平均 4.1 次 Verifier 介入 + 2.4 次 Oracle 评估）。

Framework Overview

任务环境被形式化为 POMDP $(\mathcal{X}, \mathcal{A}, T, \mathcal{O}, \Omega, R)$，Agent 目标是发现最优 Skill $S^* = \arg\max_S J(S)$，其中 $J(S)$ 是在 Skill $S$ 指导下策略 $\pi_\theta$ 到达终态时的期望奖励。由于 Ground-Truth 奖励 $R$ 仅返回不透明信号，EvoSkills 引入代理奖励 $\tilde{R}(x, V) = \frac{1}{|V|} \sum_k \mathbf{1}[e_k(x)]$（Surrogate Verifier 测试通过率）作为可优化的代理目标，通过交替优化（Skill 细化 + 测试升级）逼近真实目标。

Figure 2. 5 轮演化过程中 Skill 质量提升曲线——EvoSkills 在第 5 轮已超越人工 Skill，演化收益递增且收敛迅速。

Skill Generator

使用持久对话上下文 $C$，初始化为（任务指令 $I$，元技能 $S_{\text{meta}}$）。每次迭代读取当前 Skill $S^{(i)}$ 与累积 Verifier 诊断 $F^{(i,j)}$，生成改进版本：

$S^{(i+1)} \sim \pi_\theta(\cdot \mid S^{(i)},\, C^{(i+1)}),\quad C^{(i+1)} = C^{(i)} \oplus F^{(i,j)}$

上下文容量上限 $\beta = 0.7$（防止 LLM 上下文溢出），超限则终止当前演化轮次。Generator 执行 Skill 并产出输出文件 $x^{(i)} = \Phi(S^{(i)}, E)$，传递给 Surrogate Verifier 评估。

Surrogate Verifier

运行于完全独立的 LLM 会话 $\pi_\theta^V$，仅接收任务指令 $I$ 和输出文件 $x^{(i)}$，不继承 Generator 的推理或代码——保证测试断言与 Generator 内部状态条件独立，防止"知道答案后合谋放行"。

代理奖励：$\tilde{R}(x, V) = \frac{1}{|V|} \sum_k \mathbf{1}[e_k(x)] \in [0,1]$

当 $\tilde{R} < 1$ 时，生成结构化诊断 $F^{(i,j)}$（失败断言明细 + 根因分析 + 修复建议）反馈给 Generator。

协同演化：通路 1 — Skill 细化

当 Surrogate 测试失败时触发：固定测试套件 $V^{(j)}$，将诊断 $F^{(i,j)}$ 追加至 Generator 上下文，触发定向 Skill 修复。Verifier 测试套件在此阶段锁定不变。

约 40% 的迭代由 Surrogate Verifier 独自吸收，无需调用 Oracle，节省大量评估成本。

协同演化：通路 2 — 测试升级

当 Surrogate 通过但 Oracle 失败时触发：Oracle 返回不透明 Pass/Fail 位（无测试内容）追加至 Generator 上下文；Verifier 独立强化测试套件，生成更多样、更严格的断言：$V^{(j+1)} \sim \pi_\theta^V(\cdot \mid I, x^{(i)}, V^{(j)})$。

终止条件：最多 $N=5$ 次 Oracle 评估轮次，$M=15$ 次 Verifier 重试；Oracle 满分提前退出，保留历史最优 Skill 快照。超过 60% 的任务在 2 次 Oracle 轮次内收敛。

Analysis

消融实验

Surrogate Verifier 贡献 −30.0 pp（最大单项），是框架核心；背景知识上下文有一定帮助（+18.0 pp over no-skill），但远不足以达到完整框架的水平（差距 −22.5 pp）。

Case Study：外行星凌日周期检测

人工 Skill：5 个文件 / 1,096 行文档。

EvoSkills 演化结果：1 个统一 Skill，64 行流程文档 + 142 行可执行 Python 函数，经 5 次独立测试全部通过（100%）。

关键改进点：

  • 两阶段周期搜索（宽扫描 + 窄精化）实现为带参数的两个独立函数，而非散文中的两行提示
  • 算法被明确规定（TLS），附有从 3 次失败 BLS 尝试中得出的具体理由
  • 可执行函数替代散文说明，消除每次执行时 Agent 重新实现引入的精度 Bug

演化效率分析

  • 每任务平均 4.1 次 Verifier 介入，2.4 次 Oracle 评估
  • 约 40% 的迭代由 Surrogate Verifier 独自吸收，无需调用 Oracle
  • 超过 60% 的任务在 2 次 Oracle 轮次内收敛
  • 演化成本可控；5 轮内即可超越人工 Skill

Surrogate-GT 差距（局限揭示）

Surrogate Verifier 无法复制 Oracle 的精确精度要求（如 5 位小数匹配），也无法区分自身估算误差与 Agent 答案误差。

这揭示了"Verifier 循环自证"的潜在风险：Verifier 生成的测试断言质量决定了演化上限；若 Verifier 本身存在系统性盲区，Generator 在 Surrogate 上满分并不保证 Oracle 满分。Oracle 作为权威仲裁者的角色不可替代。

Takeaways

核心结论

  1. Agent 自动演化的 Skill 在 SkillsBench 上超越人工精心编写的 Skill（71.1% vs 53.5%），证实了"机器发现的 Skill 比人类设计的更适合 Agent"这一命题。
  2. 演化 Skill 对不同模型家族具有强迁移性（+36 至 +44 pp），编码的是可复用的任务结构而非模型特定制件。
  3. 协同演化循环（而非生成提示本身）是性能提升的根本驱动力；5 轮内即可收敛，演化成本可控。

局限性

  • Verifier 循环自证风险：Surrogate Verifier 的测试质量决定演化上限——若 Verifier 存在系统性盲区（如无法检验精确数值精度），Generator 可能在 Surrogate 满分而 Oracle 失败，形成"自我欺骗"陷阱。
  • 代码未公开（截至 2026-04-22），复现需自行实现协同演化循环及 SkillsBench 评测环境。
  • 当前框架依赖 Claude-Code 执行环境，对其他 Agent 框架的适配性待验证。

对 Skill 自动合成 / 自进化方向的启发

  1. Surrogate Verifier 作为通用信号源：以独立 LLM 会话合成测试断言作为代理奖励，是绕开稀缺 Ground-Truth 反馈的可推广范式，值得在其他 Skill 生成任务中探索。
  2. Skill 质量不能靠一次生成保证：SkillsBench 原论文发现自生成 Skill 基本无效（32.0%，与无 Skill 基线几乎持平），EvoSkills 将该结论反转至 71.1%，关键差异在于迭代验证而非生成策略本身。
  3. 认知错位是 Skill 领域的系统性问题：Finance / Natural Science 等领域人工 Skill 的大幅差异证明了人类直觉与 LLM 执行逻辑之间的结构性鸿沟；自动演化通过实际执行反馈弥合这一鸿沟。
  4. Skill 迁移性表明演化产物编码了任务抽象而非模型偏好：跨 5 家提供商 6 个模型均获显著提升，为未来构建跨模型 Skill 共享市场提供了实验依据。
  5. 与单数版 EvoSkill 的路线对比：单数版（2603.02766）依赖有标注失败样本驱动 Proposer + Pareto 前沿治理；复数版彻底摒弃 Ground-Truth 标注依赖，Surrogate 自主合成反馈，适用范围更广，但对 Verifier 质量的依赖更高。
