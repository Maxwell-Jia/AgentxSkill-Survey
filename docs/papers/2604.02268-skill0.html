Abstract

当前 Agent Skill 增强范式（inference-time skill augmentation）存在三大结构性缺陷： 检索噪声（不相关或误导性技能污染上下文）、 Token 开销（注入的 Skill 内容在多轮交互中累积，如 SkillRL 每步 2.21k token）、 未真正学习（技能知识驻留在上下文而非参数中，Agent 永远无法自主行为）。

SKILL0 提出首个以 Skill 内化 为显式训练目标的 RL 框架。核心口号： "Skills at training, zero at inference." 通过上下文内强化学习（ICRL）+ 动态课程（Dynamic Curriculum），逐步将上下文注入的 Skill 转移到模型参数中，推断阶段完全不依赖外部技能。最终在 ALFWorld 上达到 87.9%（3B）/ 89.8%（7B），超越有技能增强的 SkillRL（85.5%），同时推断 token 仅 0.38k/步（SkillRL 的 5.8 倍压缩）。

Key Insight

Skill 不必永远活在上下文里——它可以成为训练过程中的临时脚手架，通过受控的课程式退火，最终内化进模型参数，推断时实现真正零依赖的自主行为。

Framework Overview

SKILL0 由三大机制协同构成： 上下文压缩（视觉编码压缩交互历史与技能）、 In-Context RL (ICRL)（融合技能提示的样本效率与 RL 探索能力）、 动态课程（线性衰减技能预算 + 有用性驱动筛选）。

技能组织

两层结构：通用 Skill（探索策略、目标追踪）+ 任务特定 Skill（领域知识、动作序列）。 目录组织 skills/{task}/{category}.md，共 $N$ 个技能文件构成 SkillBank。 训练时按在线有用性排序选 $m$ 个注入，推断时完全不用。

视觉压缩与 ICRL

将文本历史 $h_t$ 与检索技能 $\mathcal{S}$ 编码为 RGB 图像，压缩比 $c_t \in (0,1]$ 由策略自生成（非固定超参）。 复合奖励 $\tilde{r}_t = r_t + \lambda \cdot r^{\text{comp}}_t$，其中成功时 $r^{\text{comp}}_t = \ln(c_t)$（对数形式反映边际收益递减）。 ALFWorld 上 token/步从 2.21k 降至 0.38k（>5× 压缩）。

动态课程

分 $N_S$ 阶段线性衰减技能预算： $$M^{(s)} = \left\lceil N \cdot \frac{N_S - s}{N_S - 1} \right\rceil$$ 每 $d$ 步评估有/无技能时性能差 $\Delta_k$（有用性指标）， Filter & Rank & Select 只保留当前策略仍受益的技能， 逐步逼迫模型内化知识。最优配置为 $[6,3,0]$ 线性衰减。

两阶段课程操作流程

(a) 离线相关性分组（Relevance-Driven Skill Grouping）：将验证集 $N$ 个子集 $\{\mathcal{T}_k\}_{k=1}^N$ 与对应技能文件关联，基于领域对齐为每个 $\mathcal{S}_k$ 分配专属验证任务。

(b) 在线有用性驱动课程（Helpfulness-Driven Dynamic Curriculum）：每 $d$ 步评估有/无技能条件下子任务性能差 $\Delta_k$，筛选、排序、选取 top-$m$ 技能，只保留当前策略仍受益的技能，随训练进度逐步收紧预算，逼迫模型将行为锚定到参数中。

关键消融：去 Rank（随机选择）导致性能崩溃 −13.7%（至 62.9%），充分证明精准筛选是动态课程的核心而非可选项。

Figure 6. 技能有用性 $\Delta_k$ 在各子任务上呈"先升后降"特征。 早期策略无法利用技能提示，中期学会对齐，后期课程减少技能预算强制内化，有用性回落——验证了 ICRL 与课程学习的协同机制。

Analysis

技能预算衰减策略

预算配置ALFWorld（%）备注
[6,3,0] 线性衰减87.9最优
[6,6,6] 固定全预算75.6推断移除技能 −12.3%
[3,3,3] 固定低预算80.1早期探索受限

线性衰减是唯一展现正向转移的方案。固定全预算导致过度依赖：推断时移除技能后性能骤降 12.3%，内化失败。固定低预算则早期探索受限，学习不稳定。

课程三步流程消融

配置ALFWorld（%）
完整 Filter & Rank & Select87.9
去 Filter85.2

去 Rank 随机选择导致性能崩溃至 62.9%（−13.7%），这是最关键的消融结论：基于当前策略状态的有用性排序，而非向量相似度或随机抽样，是动态课程的核心。

有用性评估间隔 $d$

$d=5$ 边际收益 +0.3%，但计算开销大；$d=10$ 是最优权衡选择；$d=20$ 评估粗粒度，性能下降。 间隔过短会频繁重新排序技能，扰乱训练稳定性；过长则有用性信号滞后，无法及时淘汰已被内化的技能。

自适应压缩比的意义

压缩比 $c_t$ 由策略自生成而非固定超参，使模型学会根据任务复杂度动态调节上下文规模。 简单任务自动压缩更激进，复杂多跳任务保留更多历史——这与人类认知负荷调节吻合，是视觉压缩机制的核心创新。

Takeaways

核心结论

  1. 提出首个显式以 Skill 内化 为训练目标的 RL 框架，推断时完全零 Skill 依赖，性能反超有技能增强的 baseline。
  2. 线性衰减技能预算 + 有用性驱动筛选是平稳内化的关键；随机选择会导致 −13.7% 性能崩溃。
  3. 自适应视觉压缩将推断 token 压缩至 0.38k/步，为 SkillRL 的 1/5.8，兼顾性能与效率。
  4. Skill 有用性指标 $\Delta_k$ 呈"先升后降"规律，为技能生命周期管理提供了可量化的退场信号。

局限性

  • 视觉压缩路径依赖多模态架构（LLaVA 系），纯文本模型不可直接复用。
  • 当前 SkillBank 固定，训练完成后不再演化；在技能需持续更新的开放域场景下适应性有限。
  • ALFWorld 和 Search-QA 场景相对封闭，对复杂工具调用（如 AppWorld 数百 API）的泛化未验证。

对后续研究的启发

  1. 从"检索注入"到"内化习得"的范式转变：推翻当前 skill augmentation 对推断期技能的必要性论证，为低延迟部署开辟新路径。
  2. 技能生命周期管理：Skill 是训练过程中的临时脚手架，有用性指标 $\Delta_k$ 的"兴衰周期"可指导增量更新策略。
  3. 精准筛选优于盲目堆积：基于当前策略状态的有用性评估（不是向量相似度）是 Skill 管理的核心判据。
  4. 推断-训练分布对齐：在推断时彻底去掉 Skill 迫使训练过程对齐，避免"提示范式"中的隐性不匹配，对 Skill 进化 / 元学习有借鉴。
  5. 与 SkillRL 的互补定位：SkillRL 适于高度模块化、技能复用率高的任务；SKILL0 适于追求低延迟、自主性强的部署环境，两者可在系统层面结合。
