Abstract

Agent 在执行任务过程中积累的经验往往分散在脚本、prompt 片段和工作流日志中，缺乏统一的提取和结构化机制，导致"重复造轮子"现象普遍。现有技能库（ClawHub、SkillsMP、Skills.sh 等）主要依赖社区下载量或 GitHub stars 进行隐式质量控制，缺乏对安全性、完整性、可执行性、可维护性、成本的多维度评估；同时，已有平台将技能视为独立制件，无法自动发现技能间的相似、组合、层级和依赖关系，妨碍大规模工作流合成和检索。

SkillNet 是首个覆盖技能全生命周期的统一开放基础设施：从异构来源（执行轨迹、GitHub 仓库、PDF/PPT/Word 文档、自然语言 prompt）自动合成技能，经五维多层次评测筛选，再构建技能本体关系图谱，最终形成含 250K+ 候选、150K+ 精选技能的开放生态。Discovery-Activation-Execution 三步渐进协议供 agent 按需检索和执行，在 ALFWorld / WebShop / ScienceWorld 三个环境上相对 ReAct 基线平均奖励提升 40%、交互步数减少 30%。

Key Insight

技能不是孤岛，而是生态。覆盖"创建-评测-连接"完整生命周期的统一框架，比任何单点优化都更有价值——LLM 驱动的自动创建解决扩展成本，五维量化评测解决质量治理，技能本体关系图谱解决检索与工作流合成，三者缺一不可。

Framework Overview

SkillNet 围绕三大核心能力构建完整的技能生命周期管理：Create（创建）从异构来源自动生成结构化技能，Evaluate（评测）从五个维度量化质量，Connect（连接）通过技能本体关系图谱建模技能间结构关系。每个技能以标准化文件夹为单位组织，核心文件 SKILL.md 包含元数据（名称、描述、使用条件）和分步执行指令，格式与 Anthropic Agent Skills 开放标准高度对齐。

Create（创建）

支持四类异构数据源自动生成技能：

  • 执行轨迹 / 对话日志：Agent 任务完成记录
  • 开源 GitHub 仓库：代码库、工具包
  • 半结构化文档：PDF、PowerPoint、Word
  • 自然语言 prompt：用户直接描述

所有创建过程均由 LLM 驱动，用户可自定义底层模型；此外通过互联网爬取和社区贡献持续扩充技能池。

Evaluate（评测）

五维评测框架，每维三级评分（Good / Average / Poor）：

  • Safety：危险操作与 prompt 注入风险
  • Completeness：步骤完整性与依赖覆盖
  • Executability：LLM 评估 + 沙箱实测
  • Maintainability：模块化与可组合性
  • Cost-awareness：时延 / 算力 / API 成本

200 样本 PhD 盲审验证：MAE < 0.03，QWK ≈ 1.000（近乎完美的评分者一致性）。

Connect（连接）

自动发现四类技能间结构关系：

  • similar_to：功能等价，支持冗余检测
  • belong_to：子组件 / 原子步骤，层级结构
  • compose_with：协同调用，输出接输入
  • depend_on：前置依赖，安全执行规划

语义嵌入匹配 + LLM 关系推理混合 pipeline，构建有向带类型多关系图。

Discovery → Activation → Execution 三步渐进协议

技能通过渐进式三步协议对 agent 暴露，实现"按需加载"（lazy-loading），避免一次性注入大量 context：

Step 1 — Discovery（发现）

Agent 只读取最小化元数据（名称 + 简短描述），进行初筛，识别与当前任务潜在相关的技能集合。此阶段 token 消耗极低。

Step 2 — Activation（激活）

当某技能描述与当前任务匹配时，agent 读取完整 SKILL.md 指令，并准备所有关联资源（脚本、模板等）。这是技能进入 agent 工作上下文的时刻。

Step 3 — Execution（执行）

Agent 遵照 SKILL.md 的分步指令执行任务，可选地运行绑定代码或调用外部资产。包含代码的技能还会在沙箱环境中进行运行时正确性验证。

与 OpenClaw 框架的 lazy-loading 设计对应，显著降低上下文开销——会话初始化时只在系统 prompt 注入紧凑元数据，按需激活完整指令。

200K+ 技能仓库的构建与质量控制

  • 初始规模：250K+ 候选技能（爬取 + 自动创建 + 社区贡献）
  • 精选规模：150K+ 高质量技能（多阶段过滤后，持续扩展）
  • 去重机制：联合比较目录结构和 SKILL.md 的 MD5 哈希值
  • 过滤机制：规则验证（格式完整性）+ 模型检查（语义质量）双层过滤
  • 10 个功能大类：Development / AIGC / Research / Science / Business / Testing / Productivity / Security / Lifestyle / Other
  • 社区贡献：自动化质检 + 随机抽样人工审计双重保障

Takeaways

核心结论

  1. SkillNet 是首个覆盖技能全生命周期（创建-评测-连接）的统一开放基础设施，将碎片化 agent 经验转化为结构化、可组合的知识单元。
  2. 五维评测框架配合自动化 LLM 评估器，在 200 样本盲测中实现与人类专家近乎完美的一致性（QWK ≈ 1.000），证明大规模自动质检的可行性。
  3. 在三个环境三个骨干模型上，相对 ReAct 基线平均奖励提升 40%、交互步数减少 30%，多骨干模型一致验证。
  4. 技能本体和关系图谱使 SkillNet 成为自进化生态而非静态仓库，技能创建、评测和分析可持续协同运转。

局限性

  • 评测框架当前主要靠 GPT-4o-mini 实例化，对不同 LLM 后端的泛化性有待验证。
  • 技能关系图谱是静态的，尚未从 agent 执行轨迹动态更新，随着技能库扩大可能出现图谱漂移。
  • Executability 维度挑战最大（最多 Average 评级），说明仅靠 LLM 静态评估代码技能不够充分，沙箱验证成本较高。

对 Skill 生态基建方向的启发

  1. Discovery-Activation-Execution 协议提供了操作语义模型，将"检索"具体化为分阶段渐进信息披露，token 效率和准确性优于 RAG 单次召回。
  2. 250K 规模意义：首次证明技能仓库的工业级可行性——不只是研究原型，而是可直接部署的基础设施，与 EvoSkill（精选小库）、AutoSkill（个人级 SkillBank）形成规模梯度。
  3. 技能关系图谱 vs. 向量索引：同时支持语义向量搜索和结构化关系查询（depend_on / compose_with 等），在工作流合成、依赖解析等复杂场景中提供向量相似度无法表达的结构信息。
  4. 五维评测框架（Safety / Completeness / Executability / Maintainability / Cost-awareness）具有较好的完备性，可作为"技能质检 API"的标准接口设计参考。
  5. 沙箱 Executability 验证是关键：大规模可执行技能仓库的刚需——仅靠 LLM 静态评估代码技能是不够的。
  6. "一人公司 / 一人实验室"愿景：在 SkillNet 支撑下，技能成为知识委托和集成的主要单元，个人策划技能仓库，agent 将其组合为工作流，记忆通过经验持续精炼。
