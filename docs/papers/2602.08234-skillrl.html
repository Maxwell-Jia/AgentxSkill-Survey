Abstract

LLM Agent 的现有学习范式面临四重结构缺陷：孤立执行（每个 Episode 重头开始，无法积累跨任务知识）、原始轨迹冗余（Reflexion / ExpeL / Mem0 等方法直接存入原始轨迹，冗长噪声大）、缺乏高层行为抽象（检索到的轨迹难以提炼可迁移策略模式）、以及记忆与策略割裂（MemRL 等方法无法让技能库与策略参数协同进化）。核心缺口在于：现有方法停留在"记忆检索"层面，而非"知识抽象"层面。

SkillRL 填补此空白，通过三阶段协同进化框架：(1) 经验蒸馏——将成功与失败轨迹分别提炼为策略示范与反事实教训；(2) 层级 SkillBank 构建——组织为通用技能 $\mathcal{S}_g$ 与任务专项技能 $\bigcup_k\mathcal{S}_k$ 两层结构；(3) 递归技能演化——基于 GRPO 强化学习，在验证失败后触发 SkillBank 的自动扩充。在 ALFWorld 上达到 89.9% 成功率，以 7B 开源模型超越 Gemini-2.5-Pro（+29.6%）和 GPT-4o（+41.9%）。

Key Insight

从轨迹到技能的抽象比从轨迹到记忆的检索更有效：消融实验显示，用原始轨迹替换技能库带来高达 25% 的性能下降，而去掉动态演化仅损失 5.5%——说明抽象形式本身比动态性更关键。

Framework Overview

SkillRL 将 Agent 策略形式化为 $\pi_{\theta}(a_t \mid o_{\leq t}, d, c)$，其中 $c$ 为检索到的技能上下文，并施加长度约束 $|c| \leq L_{\max}$。SkillBank 定义为 $\mathcal{S}_g \cup \bigcup_{k=1}^{K}\mathcal{S}_k$，通过下述三阶段 pipeline 自动构建与演化。

Figure 2. SkillRL 完整流程——用基础模型收集轨迹，蒸馏为层级技能库，冷启动 SFT 使 Agent 学会使用技能，随后在 RL 训练中基于验证失败案例进行动态技能演化。

阶段一：经验蒸馏

用基础模型 $\pi_{\text{base}}$ rollout 收集成功轨迹 $\mathcal{T}^+$ 与失败轨迹 $\mathcal{T}^-$（两者均保留）。教师模型 $\mathcal{M}_T$ 对成功轨迹提取关键决策模式；对失败轨迹合成简洁的"失败教训"——包含失败点、错误推理、应有做法、防范原则四要素。实现 10–20× token 压缩，平均 prompt 长度 <1300 tokens。

阶段二：层级 SkillBank 构建

将蒸馏结果组织为两层：通用技能 $\mathcal{S}_g$（跨任务战略原则——系统性探索策略、状态管理原则等，推理时始终注入上下文）与任务专项技能 $\mathcal{S}_k$（领域知识——特定动作序列、任务约束、失败模式，按语义相似度 $\text{TopK}({\{s: \text{sim}(e_d,e_s)>\delta\}},K)$ 检索注入）。每条技能含：名称 + 策略描述 + 适用条件。

阶段三：递归技能演化 + RL

冷启动 SFT 先教会模型检索与应用技能。随后 GRPO 强化学习以二值奖励 $R_i\in\{0,1\}$ 训练策略。每轮验证后，对成功率 $\text{Acc}(C) < \delta=0.4$ 的任务类别触发演化：分层采样失败轨迹，教师模型分析未被现有技能覆盖的失败模式，提出新技能 $\mathcal{S}_{\text{new}}$，更新 $\text{SkillBank} \leftarrow \text{SkillBank} \cup \mathcal{S}_{\text{new}}$。

GRPO 目标函数

对每个任务采样 $G$ 条轨迹，按组内归一化优势 + PPO 式裁剪更新：

$$\mathcal{J}(\theta) = \mathbb{E}\!\left[\frac{1}{G}\sum_{i=1}^{G} \min\!\left(\rho_i A_i,\,\text{clip}(\rho_i,1-\epsilon,1+\epsilon)\,A_i\right) - \beta D_{\mathrm{KL}}(\pi_\theta \| \pi_{\text{ref}})\right]$$

其中 $\pi_{\text{ref}}$ 由冷启动 SFT 后的模型担任，KL 项防止策略偏离技能使用行为范式。

"用原始轨迹替换技能库"带来最大损失（高达 25%），远超去掉动态演化（5.5%）——抽象形式本身比动态性更关键。去掉冷启动 SFT 损失 20%，说明 SFT 是模型学会使用技能的必要前提。

双路轨迹利用 + token 压缩

失败轨迹是被低估的学习信号。SkillRL 将失败轨迹合成为包含"失败点 / 错误推理 / 应有做法 / 防范原则"四要素的反事实教训，而非丢弃。过去方法通常只保留成功轨迹，浪费了失败中蕴含的边界条件信息。

技能蒸馏实现 10–20× token 压缩：平均 prompt 长度 <1300 tokens（原始轨迹方案 ~1450 tokens），不损失推理质量的同时显著降低上下文负担，使技能库在有限窗口内可容纳更丰富的经验。

层级技能的互补性

通用技能负责战略层（Progressive Goal Decomposition 等），专项技能负责战术层（No Appliance Before Object 等）；案例分析显示 Agent 会同时调用两层技能协同决策。去掉任一层均有超过 11% 的显著下降，层级结构是性能的实质贡献来源。

触发式递归演化

技能演化仅在验证成功率 $\text{Acc}(C) < \delta=0.4$ 的任务类别上触发，避免对已掌握任务的无效扩张。多样性感知分层采样：失败轨迹按类别分组、按失败严重程度排序，轮询采样维持类别熵，保证演化覆盖多样场景。SkillRL 在 60 步内达到 80%+ 成功率，而静态记忆基线在 90 步才达到更低峰值。

Takeaways

核心结论

  1. SkillRL 通过经验蒸馏 + 层级 SkillBank + 递归技能演化，在不增加上下文负担的前提下将 Agent 经验转化为持续可用的结构化知识。
  2. "经验到技能"的抽象是比"经验到记忆"更有效的迁移单元：原始轨迹替换带来的性能崩溃（最大 −25%）远大于去掉动态演化（−5.5%）。
  3. 7B 开源模型通过结构化技能学习可超越参数量大一个数量级的闭源模型（GPT-4o、Gemini-2.5-Pro）。
  4. 技能库与策略的协同演化（co-evolution）是克服局部最优、持续提升性能上限的关键机制。

局限性

  • 技能演化依赖能力强大的教师模型（o3），在无法获取强教师模型的场景下框架可用性尚待验证。
  • 技能库管理缺乏冗余检测——技能数量只增不减，长期运行可能引入冲突或冗余技能。
  • 搜索增强 QA 实验 baseline 较少，结论的稳健性有待进一步分析。

对后续研究的启发

  1. 失败轨迹是被低估的学习信号：将失败归因结构化为技能（含失败点 / 错误推理 / 正确做法 / 防范原则），比单纯记录失败轨迹更具可操作性——研究失败复用方向的工作可将此作为典型范式。
  2. SFT 冷启动是 RL 与技能库结合的必要桥梁：未经 SFT 的模型无法有效利用外部技能库，仅靠 RL 探索无法自动学会"检索 + 应用"的行为范式；这对其他将技能外化为上下文信息的框架具有普遍指导意义。
  3. 技能层级（通用 + 专项）的两分法具有方法论价值：去掉任一层均有超过 11% 的显著下降，层级结构不是装饰性设计，而是性能的实质贡献来源。
  4. 技能演化应以失败驱动、类别感知：基于验证失败触发的递归演化结合分层采样，是技能库扩充不退化的重要保障，可迁移到其他在线学习系统的知识更新策略。
  5. 技能抽象带来的 token 压缩是系统级优势：10–20× 压缩比在生产环境中的实际价值不亚于性能数字本身。
