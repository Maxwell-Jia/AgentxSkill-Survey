## 针对的问题

### 痛点

- **孤立执行，无法积累**：LLM Agent 每次任务执行相互独立，无法从成功或失败的历史中学习；每个 Episode 重头开始，不携带任何跨任务知识。
- **原始轨迹冗余噪声大**：现有基于记忆的方法（Reflexion、ExpeL、Mem0 等）将原始交互轨迹直接存入外部数据库，轨迹冗长、含大量探索步与回溯动作，提取有效信息困难。
- **缺乏高层行为抽象**：直接从轨迹检索无法提炼可迁移的策略模式；模仿过去解法而不是抽取核心原则，导致泛化能力差。
- **记忆与策略割裂**：记忆增强型 RL（MemRL 等）仅用 RL 更新记忆库而冻结策略，或仅将记忆作为辅助信息，无法让技能库与策略参数协同进化。
- **上下文膨胀**：直接塞入原始轨迹导致 token 占用剧增，在有限上下文窗口下严重挤压推理空间。

### 核心差距

现有方法均停留在"记忆检索"层面，而非"知识抽象"层面。有效的经验迁移需要从原始轨迹中提炼出高层次、可复用的技能，并让技能库随策略训练动态演化——这正是 SkillRL 所填补的空白。

---

## 核心思路

> SkillRL 将 LLM Agent 的经验蒸馏为层级技能库（SkillBank），再通过"冷启动 SFT + GRPO 强化学习 + 递归技能演化"三阶段训练，使技能库与策略参数在 RL 训练过程中协同进化，在减少上下文负担的同时大幅提升任务成功率。

### 形式化定义

Agent 策略形式化为：

$$\pi_{\theta}(a_t \mid o_{\leq t}, d, c)$$

其中 $c$ 为额外上下文（包含检索到的技能），目标是最大化期望回报，同时满足上下文长度约束 $|c| \leq L_{\max}$。

SkillBank 定义为：

$$\mathcal{S}_g \cup \bigcup_{k=1}^{K} \mathcal{S}_k$$

其中 $\mathcal{S}_g$ 为通用技能集合，$\mathcal{S}_k$ 为第 $k$ 类任务的专项技能集合。

---

### Pipeline / 算法流程（Algorithm 1）

**阶段一：经验蒸馏（Experience-based Skill Distillation）**

1. 用基础模型 $\pi_{\text{base}}$ 在环境中 rollout，收集成功轨迹 $\mathcal{T}^+$ 与失败轨迹 $\mathcal{T}^-$（两者均保留，失败轨迹包含独有的边界条件信息）。
2. 对成功轨迹，教师模型 $\mathcal{M}_T$ 提取关键决策模式与可迁移策略：$s^+ = \mathcal{M}_T(\tau^+, d)$。
3. 对失败轨迹，直接放入上下文不可行（冗长且噪声大），转而合成简洁的"失败教训"：$s^- = \mathcal{M}_T(\tau^-, d)$，包含失败点、错误推理、应有做法、防范原则四要素。

**阶段二：层级技能库构建（SkillBank Construction）**

将蒸馏结果组织为两层结构：
- **通用技能 $\mathcal{S}_g$**：跨任务类型通用的战略原则，包括系统性探索策略、状态管理原则、目标追踪启发式规则等。
- **任务专项技能 $\mathcal{S}_k$**：针对具体任务类别的领域知识，包括特定动作序列、任务约束、常见失败模式与优化流程。

每条技能具有统一结构：名称（concise name）+ 策略描述（principle）+ 适用条件（when_to_apply）。

推理时，通用技能始终包含在上下文中；专项技能通过语义相似度检索：

$$\mathcal{S}_{\text{ret}} = \text{TopK}\left(\{s \in \mathcal{S}_k : \text{sim}(e_d, e_s) > \delta\}, K\right)$$

策略条件化在检索结果上：$a_t \sim \pi_\theta(a_t \mid o_{\leq t}, d, \mathcal{S}_g, \mathcal{S}_{\text{ret}})$

**阶段三：递归技能演化 + RL 训练（Recursive Skill Evolution via RL）**

- **冷启动 SFT**：基础模型尚不会使用技能；教师模型生成 $N$ 条技能增强推理示范 $\mathcal{D}_{\text{SFT}} = \{(d_i, \mathcal{S}_i, \tau_i^*)\}$，对基础模型微调得到 $\pi_{\theta_{\text{sft}}}$，同时作为后续 KL 正则化的参考策略 $\pi_{\text{ref}}$。
- **GRPO RL 训练**：对每个任务采样 $G$ 条轨迹，用二值奖励 $R_i \in \{0, 1\}$ 计算组内归一化优势，按 PPO 式裁剪目标更新策略：

$$\mathcal{J}(\theta) = \mathbb{E}\left[\frac{1}{G}\sum_{i=1}^{G} \min\left(\rho_i A_i, \text{clip}(\rho_i, 1-\epsilon, 1+\epsilon) A_i\right) - \beta D_{\text{KL}}(\pi_\theta \| \pi_{\text{ref}})\right]$$

- **递归技能演化**：每轮验证后，对成功率 $\text{Acc}(C) < \delta$ 的任务类别，用分层采样策略收集失败验证轨迹，由教师模型分析未被现有技能覆盖的失败模式，提出新技能或改进方案：$\mathcal{S}_{\text{new}} = \mathcal{M}_T(\mathcal{T}_{\text{val}}^-, \text{SkillBank})$，更新库 $\text{SkillBank} \leftarrow \text{SkillBank} \cup \mathcal{S}_{\text{new}}$。

---

## 关键设计细节

- **双路轨迹利用**：成功轨迹提炼为策略示范，失败轨迹合成为反事实教训（counterfactuals）。过去方法通常只保留成功轨迹，浪费了失败中蕴含的边界条件信息。
- **10-20× token 压缩**：技能蒸馏相比原始轨迹实现 10-20 倍的 token 压缩，且不损失推理质量；实测平均 prompt 长度 <1300 tokens，对比原始记忆方案的 ~1450 tokens，减少约 10.3%。
- **冷启动 SFT 的必要性**：直接向未经训练的模型提供技能效果有限；SFT 阶段教会模型如何检索、解读、应用技能，是 RL 阶段能发挥作用的前提。
- **触发式演化（threshold-based）**：技能演化仅在验证成功率低于阈值 $\delta = 0.4$ 的任务类别上触发，避免对已掌握任务的无效扩张。
- **多样性感知分层采样**：失败轨迹按类别分组，按失败严重程度（负奖励大小）排序，轮询采样维持类别熵，保证演化覆盖多样场景。
- **双层技能的互补性**：通用技能提供基础战略框架，专项技能处理细粒度领域约束；案例分析显示 Agent 会同时调用"Progressive Goal Decomposition"（通用）与"No Appliance Before Object"（专项）等技能协同决策。

## 实验与效果

### 数据集 / Benchmark

- **ALFWorld**：文本 household 任务游戏，包含 Pick / Look / Clean / Heat / Cool / PickTwo 六类子任务，考察导航与物品操作能力。
- **WebShop**：Web 购物环境模拟，Agent 需导航网页找到符合描述的商品并完成购买。
- **搜索增强 QA**：7 个数据集，含单跳（NQ、TriviaQA、PopQA）与多跳（HotpotQA、2WikiMultiHopQA、MuSiQue、Bamboogle），训练在 NQ + HotpotQA，测试包含 OOD 迁移。
- **基础模型**：Qwen2.5-7B-Instruct；**教师模型**：OpenAI o3（用于技能蒸馏和 SFT 数据生成）。

### 主结果（Table 1：ALFWorld & WebShop）

| 方法 | ALFWorld（全） | WebShop（成功率） |
|------|--------------|-----------------|
| GPT-4o | 48.0% | 23.7% |
| Gemini-2.5-Pro | 60.3% | 35.9% |
| Reflexion | 42.7% | 28.8% |
| GRPO（基线） | 77.6% | 66.1% |
| Mem0+GRPO | 54.7% | 37.5% |
| SimpleMem+GRPO | 62.5% | 46.9% |
| EvolveR | 43.8% | 17.6% |
| MemRL | 21.4% | 9.2% |
| **SkillRL** | **89.9%** | **72.7%** |

- SkillRL 在 ALFWorld 上比 Gemini-2.5-Pro 高出 29.6%，比 GPT-4o 高出 41.9%，用 7B 开源模型超越闭源大模型。
- 相比 vanilla GRPO，绝对提升 **12.3%**；在最难子任务 Cool（+23.0%）和 PickTwo（+22.8%）提升最大。
- 相比最强记忆增强 RL 基线 Mem0+GRPO，ALFWorld 高出约 **35.2%**。

### 搜索增强 QA 结果（Table 2）

| 方法 | 平均分 |
|------|--------|
| Search-R1 | 38.5% |
| EvolveR | 43.1% |
| **SkillRL** | **47.1%** |

SkillRL 在 Bamboogle 上比 EvolveR 高出 19.4%；在 OOD 任务（TriviaQA、2Wiki）上保持竞争力，证明搜索策略技能具有跨任务泛化能力。

### 消融实验（Table 3，文中数字）

| 消融项 | ALFWorld 降幅 | WebShop 降幅 |
|--------|-------------|-------------|
| 去除通用技能（仅保留专项技能） | -13.1% | -11.3% |
| 用原始轨迹替换技能库 | 最大 -25% | — |
| 去掉冷启动 SFT | -20% | — |
| 去掉动态技能演化 | -5.5% | — |

### 技能库演化

训练初始：55 个技能（12 通用 + 43 专项）；训练结束（Step 150）：100 个技能（20 通用 + 80 专项），增量主要来自专项技能的持续扩充。

## 结论与启发

### 论文结论

- SkillRL 通过三个核心机制——经验蒸馏、层级 SkillBank 构建、递归技能演化——在不增加上下文负担的前提下，让 Agent 的经验转化为持续可用的结构化知识。
- "经验到技能"的抽象是比"经验到记忆"更有效的迁移单元：消融显示，用原始轨迹替换技能库的下降幅度（高达 25%）远超去掉动态演化（5.5%），说明抽象本身比动态性更关键。
- 小规模开源模型（7B）通过结构化技能学习可以超越参数量大一个数量级的闭源模型。
- 技能库与策略的协同演化（co-evolution）是克服局部最优、持续提升性能上限的关键：SkillRL 在 60 步内达到 80%+ 成功率，而静态记忆基线在 90 步才达到更低峰值。

### 对 Skill 自动合成 / 自进化方向的启发

1. **失败轨迹是被低估的学习信号**：SkillRL 将失败轨迹蒸馏为反事实教训而非丢弃，这一设计体现了"失败即数据"的核心思想；综述中研究失败复用的工作应将此作为典型案例。将失败归因分析（识别失败点、错误推理、正确做法、防范原则）结构化为技能，比单纯记录失败轨迹更具可操作性。

2. **SFT 冷启动是 RL 与技能库结合的必要桥梁**：未经 SFT 初始化的模型无法有效利用外部技能库，仅靠 RL 探索无法自动学会"检索+应用"技能的行为范式；这对其他将技能外化为上下文信息的框架具有普遍指导意义。

3. **技能层级（通用 + 专项）的两分法具有方法论价值**：通用技能负责战略层、专项技能负责战术层，二者互补而非替代；去掉任一层都有超过 11% 的显著下降，说明层级结构不是装饰性设计，而是性能的实质贡献来源。

4. **技能演化应以失败驱动、类别感知**：基于验证失败触发的递归演化避免了对已掌握任务的无效更新；结合分层采样维持类别多样性，是技能库扩充不退化的重要保障——可迁移到其他在线学习系统的知识更新策略。

5. **技能抽象带来的 token 压缩是系统级优势**：10-20× 的压缩比不仅降低推理成本，还使技能库能在有限上下文中容纳更丰富的经验；在长上下文代价高昂的生产环境中，这一特性的实际价值不亚于性能数字本身。

6. **局限性**：本文技能演化依赖能力强大的教师模型（o3），在无法获取强教师模型的场景下框架的可用性尚待验证；技能库管理缺乏冗余检测（技能数量只增不减），长期运行可能引入冲突或冗余技能；此外搜索增强 QA 实验的基础模型规模与 ALFWorld/WebShop 相同但 baseline 更少，结论的稳健性有待进一步分析。
