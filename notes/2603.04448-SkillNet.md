## 针对的问题

### 痛点

1. **技能孤岛问题**：Agent 在执行任务过程中积累的经验分散在脚本、prompt 片段或工作流日志里，缺乏统一的提取和结构化机制，导致"重复造轮子"现象普遍。
2. **无系统质量保证**：现有技能库（ClawHub、SkillsMP、Skills.sh 等）主要依赖社区下载量或 GitHub stars 进行隐式质量控制，缺乏对安全性、完整性、可执行性、可维护性、成本的多维度评估。
3. **技能孤立、关系缺失**：已有平台将技能视为独立制件，无法自动发现技能间的相似、组合、层级和依赖关系，妨碍大规模工作流合成和检索。
4. **来源单一、创建手工**：主流技能仓库不支持从 agent 执行轨迹、GitHub 仓库、PDF/PPT/Word 文档等异构来源自动生成技能，全靠人工手写，扩展成本极高。
5. **验证不透明、引入技术债**：质量评估只在下游任务表现中间接体现，stochastic 且不可解释，随着技能库增大将形成系统性技术债。

### 核心差距

当前 AI 基础设施在技能层面缺少一个能覆盖"创建-评测-连接"完整生命周期的统一框架：要么只有静态包管理（npm-like），要么只有社区市场（marketplace），没有系统能同时支持 LLM 驱动的自动创建、多维量化评测和关系图谱级分析。SkillNet 正是为填补这一缺口而设计的。

---

## 核心思路

> SkillNet 是一套覆盖技能全生命周期的开放基础设施：从异构来源自动合成技能 → 五维多层次评测筛选 → 构建技能本体关系图谱，最终形成一个含 250K+ 候选、150K+ 精选技能的开放生态，并通过 Discovery-Activation-Execution 三步渐进协议供 agent 按需检索和执行。

## 方法

### 整体框架

![Figure 2: SkillNet 端到端 Pipeline](https://arxiv.org/html/2603.04448v1/x3.png)

**Figure 2**：SkillNet 端到端 Pipeline。系统将用户输入（执行轨迹、GitHub 项目、文档、prompt）和互联网资源转化为可执行技能，经过多维评测后组织为结构化技能网络，支持搜索、下载、分析和社区贡献。

### Skill 格式规范

SkillNet 沿用并扩展了 Anthropic 在 agentskills.io 上发布的 Agent Skills 开放标准（论文引用：Anthropic [2025] "Agent skills: An open standard for modular ai agent capabilities"）。每个技能以**结构化文件夹**为单位组织：

- **SKILL.md**：核心文件，包含元数据（名称、描述、使用条件）和分步执行指令
- **可选附件**：辅助脚本（Python / TypeScript）、模板、文档、参考资料等

SKILL.md 的元数据字段设计与 Anthropic 官方规范高度对齐：名称（name）、描述（description）、使用条件（usage conditions）是三个必填元素；SkillNet 在此基础上通过 Skill Ontology 额外引入分类标签（category / tag）和关系类型（similar_to / belong_to / compose_with / depend_on），作为仓库级管理的扩展字段，而非修改单个 SKILL.md 的核心格式。换言之，SkillNet 的技能格式与 Anthropic 规范完全兼容，差异在于 SkillNet 在仓库层多了一层本体结构和关系图谱。

### Discovery → Activation → Execution 三步协议

技能通过渐进式三步协议对 agent 暴露，实现"按需加载"（lazy-loading），避免一次性注入大量 context：

**Step 1 — Discovery（发现）**

Agent 只读取最小化元数据（名称 + 简短描述），对候选技能进行初筛，识别与当前任务潜在相关的技能集合。此阶段 token 消耗极低。

**Step 2 — Activation（激活）**

当某个技能的描述与当前任务匹配时，agent 读取该技能的完整 SKILL.md 指令，并准备所有关联资源（脚本、模板等）。这是技能进入 agent 工作上下文的时刻。

**Step 3 — Execution（执行）**

Agent 遵照 SKILL.md 中的分步指令执行任务，可选地运行绑定的代码或调用引用的外部资产来完成任务。包含代码或工具调用的技能还会在沙箱环境中进行运行时正确性验证。

这一协议与 OpenClaw 框架的 lazy-loading 设计完全对应：会话初始化时只在系统 prompt 注入紧凑的元数据，按需激活完整指令，显著降低上下文开销。

### 三大子能力

![Figure 3: SkillNet Skill Ontology](https://arxiv.org/html/2603.04448v1/x4.png)

**Figure 3**：SkillNet Skill Ontology，三层结构：Skill Taxonomy（分类层）→ Skill Relation Graph（关系层）→ Skill Package Library（包层）。

#### Create（创建）

SkillNet 支持四类异构数据源自动生成技能：

| 来源类型 | 示例 |
|---------|------|
| 执行轨迹与对话日志 | Agent 任务完成记录、交互 log |
| 开源 GitHub 仓库 | 代码库、工具包 |
| 半结构化文档 | PDF、PowerPoint、Word |
| 自然语言 prompt | 用户直接描述 |

此外，SkillNet 通过互联网爬取、内部开发和社区贡献持续扩充技能池。所有创建过程均由 LLM 驱动，用户可自定义底层模型。

#### Evaluate（评测）

多维评测框架覆盖五个核心维度，每维度三级评分（Good / Average / Poor）：

| 维度 | 评测内容 |
|------|---------|
| **Safety（安全性）** | 检测危险操作（未授权文件删除等）和 prompt 注入风险 |
| **Completeness（完整性）** | 是否包含所有必要步骤、前提条件、依赖和执行约束 |
| **Executability（可执行性）** | LLM 评估 + 沙箱实际运行验证，识别幻觉工具调用或模糊指令 |
| **Maintainability（可维护性）** | 模块化和可组合性，局部更新不破坏全局依赖 |
| **Cost-awareness（成本感知）** | 量化时间延迟、计算资源消耗和 API 调用成本 |

评测主要由自动化 LLM 评估器（GPT-4o-mini 实例化）执行，辅以细粒度打分 rubric。可执行性维度额外引入沙箱实测。

评测器可靠性经 200 个随机抽样技能、3 位 PhD 级盲审验证：MAE < 0.03，QWK ≈ 1.000（近乎完美的评分者一致性）。

#### Connect（连接）

Skill Analysis 模块自动发现并建模技能间结构关系，构建大规模技能关系图谱：

| 关系类型 | 语义 |
|---------|------|
| **similar_to** | 功能等价或高度相似，支持冗余检测和替换 |
| **belong_to** | 子组件或原子步骤，捕获层级结构 |
| **compose_with** | 协同调用，一个技能的输出通常是另一个的输入 |
| **depend_on** | 前置依赖，支持安全执行规划 |

关系图谱通过语义嵌入匹配 + LLM 关系推理混合 pipeline 自动构建，形成有向带类型多关系图。

## 关键设计细节

### 200K+ 技能制品仓库的构建

- **初始规模**：250K+ 候选技能，来自互联网爬取、自动创建 pipeline 和社区贡献
- **精选规模**：经多阶段过滤后，150K+ 高质量技能进入最终仓库（持续扩展）
- **去重机制**：联合比较技能目录结构和 SKILL.md 的 MD5 哈希值，有效消除冗余
- **过滤机制**：规则验证（格式完整性）+ 模型检查（语义质量）双层过滤
- **分类标签**：10 个功能大类（Development / AIGC / Research / Science / Business / Testing / Productivity / Security / Lifestyle / Other）+ 细粒度语义标签
- **社区贡献**：社区提交的技能同样经过相同的自动化质量检查；此外通过随机抽样进行定期人工审计

### Python Toolkit（skillnet-ai）

![Figure 5: skillnet-ai 工具包用法示例](https://arxiv.org/html/2603.04448v1/x6.png)

**Figure 5**：skillnet-ai 工具包实际使用示例，展示 CLI 命令行（左）和 Python 库调用（右）两种接口形态。

skillnet-ai 提供统一的 CLI + Python 库双接口：

| 功能 | CLI 命令 | 说明 |
|------|---------|------|
| 关键词 / 语义搜索 | `skillnet search` | 向量检索 + 关键词匹配 |
| 下载技能 | `skillnet download` | 直接从 GitHub 下载到本地工作区 |
| 创建技能 | `skillnet create` | 从轨迹、GitHub repo、文档或 prompt 生成结构化技能 |
| 评测技能 | `skillnet evaluate` | 五维度评分 |
| 分析关系 | `skillnet analyze` | 生成技能关系图谱报告 |

API 接口：支持关键词搜索和向量搜索，端点为 `http://api-skillnet.openkg.cn/v1/search`。

### Progressive Disclosure 设计

Discovery-Activation-Execution 三步协议是渐进式信息披露（progressive disclosure）在 Agent 技能管理中的直接应用：只在必要时加载完整上下文，将 token 消耗集中在真正相关的技能上。这与 OpenClaw 框架的 lazy-loading 实践一致，也与 Anthropic CLAUDE.md 机制中的"按需启用"思想相通。

### Skill Ontology 的动态性

技能本体是动态的：分类层（Taxonomy）的标签可持续添加，LLM 从标签推断关系，实现技能关系图谱的持续实例化。SkillNet 因此是自进化生态而非静态集合。

### 与 MCP 的关系

论文未直接讨论 MCP（Model Context Protocol），但 SkillNet 的 SKILL.md 格式和 Python toolkit 已为 MCP 风格的工具接入提供结构基础：技能可被视为轻量级、自描述的"工具包"，其元数据可直接映射到 MCP tool schema 中的 name / description / inputs 字段。

## 实验与效果

### 评测环境

三个文本仿真环境，均建模为 POMDP（部分可观测马尔可夫决策过程）：

| 环境 | 任务类型 |
|------|---------|
| **ALFWorld** | 家居具身环境，导航和物体操作 |
| **WebShop** | 在线购物仿真，产品搜索、比较、购买 |
| **ScienceWorld** | 虚拟科学实验室，科学仪器操作和实验执行 |

骨干模型：DeepSeek V3.2、Gemini 2.5 Pro、o4 Mini。基线方法：ReAct、ExpeL、Few-Shot。

### 主结果

![Figure 6: 各方法性能对比](https://arxiv.org/html/2603.04448v1/x7.png)

**Figure 6**：SkillNet 在三个环境、三个骨干模型上的性能对比，一致超越 ReAct 和 Few-shot 基线，平均奖励（R）显著提升，交互步数（S）显著降低。

**Table 1**（主要数值结果节选）：

| 模型 | 方法 | ALFWorld Seen R | ALFWorld Unseen R | WebShop R | ScienceWorld Seen R | ScienceWorld Unseen R |
|------|------|----------------|------------------|-----------|--------------------|-----------------------|
| DeepSeek V3.2 | ReAct | 66.43 | 69.40 | 31.55 | 69.86 | 64.67 |
| | + SkillNet | **80.60** | **83.57** | **46.18** | **84.87** | **81.31** |
| Gemini 2.5 Pro | ReAct | 60.00 | 61.94 | 31.66 | 58.24 | 56.13 |
| | + SkillNet | **91.43** | **91.04** | **53.02** | **88.84** | **86.26** |
| o4 Mini | ReAct | 45.71 | 49.25 | 24.19 | 64.89 | 59.93 |
| | + SkillNet | **68.57** | **73.28** | **36.21** | **73.24** | **71.06** |

核心指标（相对 ReAct 平均）：**平均奖励提升 40%，交互步数减少 30%**。

### 性能分析

- **各规模模型均受益**：从紧凑模型（o4 Mini，+15.7R）到大规模模型（Gemini 2.5 Pro，+28.5R），性能提升随模型能力变化但始终显著，表明 SkillNet 提供的是超越参数知识的互补能力。
- **Seen/Unseen 均有增益**：在 Seen 和 Unseen 测试分割上的一致收益表明，技能抽象和复用能够促进跨任务知识迁移，不依赖于记忆特定实例。
- **技能来源**：评测所用技能集合从 ETO 专家轨迹合成，训练集和测试集无重叠，排除数据泄露。

## 结论与启发

### 论文结论

1. SkillNet 是首个覆盖技能全生命周期（创建-评测-连接）的统一开放基础设施，将碎片化 agent 经验转化为结构化、可组合的知识单元。
2. 五维评测框架配合自动化 LLM 评估器，在 200 样本盲测中实现与人类专家近乎完美的一致性（QWK ≈ 1.000），证明大规模自动质检的可行性。
3. 在 ALFWorld、WebShop、ScienceWorld 三个环境上，相对 ReAct 基线平均奖励提升 40%、交互步数减少 30%，多骨干模型一致验证。
4. 技能本体和关系图谱使 SkillNet 成为自进化生态而非静态仓库，技能创建、评测和分析可持续协同运转。

### 对 Skill 管理 / 检索 / 生态方向的启发

1. **Discovery-Activation-Execution 协议对综述"技能生命周期"一节的意义**：这三步协议提供了一个清晰的操作语义模型，将"检索"（retrieval）从通用概念具体化为分阶段的渐进信息披露，与 RAG 的单次召回相比在 token 效率和准确性上均有改进。综述在讨论"技能检索"时应将此作为重要实现范式引用。

2. **250K 规模的含义**：SkillNet 是目前公开的规模最大的结构化 Agent 技能仓库，且 150K+ 经过五维质检。这一规模首次证明技能仓库的工业级可行性——不只是研究原型，而是可直接部署的基础设施。与 EvoSkill（自动进化小规模精选库）、AutoSkill（个人用户级 SkillBank）形成规模梯度对比。

3. **技能关系图谱 vs. 向量索引**：SkillNet 同时支持语义向量搜索和结构化关系查询（depend_on / compose_with 等），是 flat 向量库的重要进化方向——在工作流合成、依赖解析等复杂场景中，关系图谱能提供向量相似度无法表达的结构信息。

4. **五维评测框架是质量治理的参考标准**：Safety / Completeness / Executability / Maintainability / Cost-awareness 五个维度的设计具有较好的完备性，且已获人类专家验证。未来构建技能生态时，这五个维度可作为"技能质检 API"的标准接口，而非每个系统各自定义评测指标。

5. **沙箱 Executability 验证是关键**：论文明确指出 Executability 在五维中挑战最大（最多 Average 评级），而这一维度引入了沙箱实测作为 LLM 评估的补充。这说明仅靠 LLM 静态评估代码技能是不够的，沙箱验证是大规模可执行技能仓库的刚需。

6. **"一人公司 / 一人实验室"的设想**：论文 Conclusion 明确提出，在 SkillNet 的支撑下，技能成为知识委托和集成的主要单元——个人策划技能仓库，agent 将其组合为工作流，记忆通过经验持续精炼。这是对综述"Skill 生态基建"方向的重要愿景背书，值得在综述的展望章节引用。
